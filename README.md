# Artemis Orchestration App

로컬 실행기와 외부 모델 연결을 한 화면에서 다루기 위해 만든 AI 오케스트레이션 웹 애플리케이션입니다.  
채팅, 로컬 파일 작업, 공개 피드 수집, 오케스트레이션 시각화, 에이전트 설정을 한 프로젝트 안에서 실험하고 있습니다.

## 프로젝트 소개

이 프로젝트는 다음 문제를 해결하기 위해 만들고 있습니다.

- 여러 AI 모델과 실행기를 한 UI에서 선택하고 실행하기
- 로컬 작업 폴더와 연결된 파일 흐름을 관리하기
- 공개 정보 피드를 수집해 시그널 형태로 정리하기
- 에이전트, 모델, 스킬, 출력 흐름을 오케스트레이션 화면에서 시각적으로 확인하기

단순한 데모 화면이 아니라, 실제 실행기와 브리지를 붙여 로컬 환경에서 동작하는 구조를 목표로 개발 중입니다.

## 현재 구현 기능

- 채팅
  - Codex CLI, Ollama 기반 로컬 실행
  - OpenAI 호환 API, Gemini, Claude 연결을 위한 설정 구조
  - 선택한 실행기/모델 기준으로 응답 요청
- 내 파일
  - 로컬 작업 폴더 기준 파일 목록 조회
  - 업로드 및 새 폴더 생성
  - 저장된 파일 미리보기
- 시그널
  - 공개 피드 기반 실시간 항목 수집
  - 워치리스트 저장
  - 카테고리별 필터
- 오케스트레이션
  - 에이전트, 모델, 메모리, 스킬, 출력 흐름을 노드 형태로 시각화
  - 실행 로그 확인
- 설정
  - 에이전트 프로필 관리
  - API 키 저장
  - 실행기 연결 상태 확인
  - 브리지 URL 및 환경 설정

  - ESLint

## 실행 방법

### 1. 요구 사항

- Node.js 20 이상
- npm
- 선택 사항
  - Ollama
  - Codex CLI

### 2. 설치

```bash
npm install
```

### 3. 브리지 서버 실행

```bash
npm run bridge
```

기본 주소는 `http://127.0.0.1:4174` 입니다.

### 4. 프론트엔드 실행

개발 서버:

```bash
npm run dev
```

또는 빌드 후 미리보기:

```bash
npm run build
npm run preview
```

기본 프론트 주소는 `http://127.0.0.1:4173` 또는 Vite가 안내하는 주소입니다.
