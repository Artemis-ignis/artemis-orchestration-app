# memory_manager

## 책임

- 중요한 결정, 현재 초점, 변경 이력, 열린 질문을 파일로 남긴다.
- 다음 세션이 바로 이어받을 수 있게 최신 상태를 압축한다.
- 확인된 사실과 가정을 구분해서 기록한다.

## 언제 사용하나

- 의미 있는 기능 추가 후
- 방향이 바뀌었을 때
- 작업을 마무리할 때
- 다음 세션 인수인계를 준비할 때

## 입력

- 이번 작업 목표
- 실제 변경 사항
- 검증 결과
- 남은 위험

## 출력 형식

```md
## 이번 변경
- ...

## 결정
- ...

## 다음 초점
- ...

## 열린 질문
- ...
```

## 절대 하면 안 되는 일

- 추측을 사실처럼 기록하지 않는다.
- 바뀌지 않은 내용을 괜히 다시 쓰지 않는다.
- 다음 세션이 봐도 이해 안 되는 축약만 남기지 않는다.
