# Artemis Review 기준

리뷰 순서는 항상 아래를 따른다.

1. 기능 회귀
2. 상태 흐름 일관성
3. 정보 구조와 시각 위계
4. 접근성
5. 유지보수성

## 기본 체크

- 기존 라우팅, 브리지 호출, 저장 흐름을 깨지 않았는가
- 한 화면에서 같은 역할의 버튼, 카드, 입력, 상태 표현이 같은 규칙을 따르는가
- 전역 CSS 임시 덮어쓰기보다 토큰/프리미티브/페이지 계층에서 해결했는가
- 빈 상태, 로딩, 오류, 준비 안 됨 상태가 모두 실제로 보이는가
- 모바일/태블릿/데스크톱에서 잘림, 과밀, 의미 없는 공백이 없는가
- 상태 문구가 연결 상태와 실행 상태를 혼동하지 않는가

## UI 품질 기준

- 한 화면에 주인공 작업면이 하나만 보여야 한다
- 보조 정보는 패널, 배지, 디밈 텍스트로 낮춰야 한다
- 강조색은 행동 또는 상태에만 써야 한다
- 카드가 꼭 필요하지 않으면 레이아웃으로 해결한다
- 배너는 실제 조치가 필요한 경우에만 쓴다

## 리팩터링 기준

- 400줄 이상 파일은 분리 후보로 본다
- 공용 패턴이 2회 이상 반복되면 프리미티브로 승격한다
- 서버 파일은 거대한 조건문을 늘리기보다 기능 단위 핸들러로 빼는 쪽을 우선 검토한다
