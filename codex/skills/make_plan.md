# make_plan

## 목적

수정 전에 짧고 검증 가능한 계획을 만든다.

## 실행 절차

1. 목표를 한 문장으로 적는다.
2. 성공 기준을 적는다.
3. 작업을 3~6단계로 나눈다.
4. 각 단계마다 검증 방법을 붙인다.

## 출력 템플릿

```md
## 목표
- ...

## 성공 기준
- ...

## 단계
1. ...
   검증: ...
2. ...
   검증: ...
3. ...
   검증: ...
```

## 금지

- "전부 정리" 같은 모호한 단계만 적지 않는다.
- 검증이 없는 단계를 두지 않는다.
