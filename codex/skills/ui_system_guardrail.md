# UI System Guardrail

목표: 화면을 고칠 때마다 제품 전체의 시각 체계를 더 단단하게 만든다.

## 반드시 지킬 것

- 색, 반경, 그림자, 간격은 토큰만 사용한다
- 화면을 직접 박스들로 꾸미지 말고 공용 프리미티브를 먼저 조합한다
- 페이지별 특별 규칙은 공용 규칙 위에 얹는 최소 오버라이드로 끝낸다
- 같은 의미의 상태는 같은 톤과 문구 체계를 쓴다

## 금지

- 즉흥적인 hex 색상 추가
- one-off 패딩, 반경, 그림자
- 같은 역할의 버튼에 화면마다 다른 높이와 두께를 쓰는 것
- 전역 CSS 맨 아래에 임시 `!important` 누적

## 점검 질문

- 이 변경이 다른 페이지에도 바로 재사용 가능한가
- 카드가 아니라 레이아웃으로 해결할 수 없는가
- 강조가 너무 많아 주작업면이 흐려지지 않는가
