# safe_edit

## 목적

작은 단위로 수정하고 즉시 검증 가능한 상태를 유지한다.

## 실행 절차

1. 수정할 파일을 명시한다.
2. 변경 이유를 한 줄로 적는다.
3. `apply_patch`로 필요한 줄만 수정한다.
4. 본인 변경으로 생긴 미사용 코드만 정리한다.
5. 수정 직후 최소 검증을 실행한다.

## 출력 템플릿

```md
## 수정 파일
- ...

## 변경 이유
- ...

## 즉시 검증
- ...
```

## 금지

- 한 번에 unrelated 파일을 여러 개 건드리지 않는다.
- "겸사겸사" 리팩터링을 넣지 않는다.
- 검증 전에 다음 큰 단계로 넘어가지 않는다.
