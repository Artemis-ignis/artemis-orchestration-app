# run_verification

## 목적

변경에 맞는 가장 작은 충분 조건 검증을 실행하고 기록한다.

## 우선순위

1. 타입/문법 검증
2. 린트
3. 빌드
4. 단위 테스트
5. 헤드리스 UI 확인

## 실행 절차

1. 변경 종류를 분류한다.
2. 아래 중 맞는 조합을 고른다.

- 문서/메모리만 변경: 링크, 경로, 참조 확인
- 프런트 UI 변경: `npm run lint`, `npm run build`, 헤드리스 화면 확인
- 브리지/API 변경: `npm run lint`, `npm run build`, 관련 엔드포인트 확인
- 라우팅/선택 로직 변경: 관련 테스트 또는 실제 최소 재현

3. 결과를 그대로 기록한다.

## 출력 템플릿

```md
## 실행한 검증
- 명령: ...
- 결과: ...

## 실제로 확인한 것
- ...

## 미확인 항목
- ...
```

## 금지

- 실패를 숨기지 않는다.
- UI 작업인데 화면 확인 없이 끝내지 않는다.
