# inspect_task

## 목적

작업을 시작하기 전에 관련 범위를 좁히고, 확인된 사실과 가정을 분리한다.

## 실행 절차

1. 루트 구조를 확인한다.
2. 요청과 직접 연결되는 파일만 찾는다.
3. 관련 파일을 읽고, 현재 동작을 요약한다.
4. 아래 형식으로 기록한다.

```md
## 확인된 사실
- ...

## 가정
- ...

## 영향 범위
- 파일:
  - ...

## 바로 다음 액션
- ...
```

## 기본 체크

- 기존 `HANDOFF.md`가 있으면 먼저 읽는다.
- 기존 협업 문서나 메모리가 있으면 재사용한다.
- UI 작업이면 실제 렌더 경로와 스타일 파일을 함께 확인한다.
- 서버 작업이면 저장소, 상태, 라우터, API 경계를 먼저 확인한다.

## 금지

- 파일을 읽지 않고 구조를 추정하지 않는다.
- 영향 범위를 넓게 잡아 불필요한 수정으로 이어가지 않는다.
