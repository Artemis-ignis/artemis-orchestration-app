# Page Refactor To Primitives

목표: 큰 페이지 파일을 공용 프리미티브 중심 구조로 재편한다.

## 순서

1. 페이지의 주작업면, 보조 컨텍스트, 도구 영역을 나눈다
2. 공통 헤더, 섹션 헤더, 패널, 상태 배지, 툴바로 치환한다
3. 데이터 로직은 유지하고 마크업만 먼저 정리한다
4. 반복 블록이 보이면 하위 섹션 컴포넌트로 분리한다

## 기준

- 페이지 파일은 상태 로직과 레이아웃 조합에 집중시킨다
- 카드 안에 카드가 반복되면 구조를 다시 본다
- 상태 배너보다 인라인 상태 설명을 우선한다
- 결과 확인 위치는 항상 고정돼 있어야 한다
